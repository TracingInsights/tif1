The `io_pipeline` module contains internal helpers for transforming raw JSON data into structured DataFrames. These functions handle column renaming, type coercion, and data validation.

<Warning>
  This module contains internal implementation details. The API is subject to change. Most users should use the high-level `Session` API instead.
</Warning>

## Overview

The I/O pipeline follows this flow:

```mermaid
graph LR
    A[Raw JSON] --> B[_validate_json_payload]
    B --> C[_extract_driver_info_map]
    B --> D[_create_lap_df]
    D --> E[_process_lap_df]
    E --> F[Final DataFrame]
```python

### `_create_lap_df`

```python
def _create_lap_df(
    lap_data: list[dict[str, Any]],
    lib: Literal["pandas", "polars"]
) -> DataFrame
```python

Create a DataFrame from raw lap data JSON.

**Parameters:**
- `lap_data`: List of lap dictionaries from JSON
- `lib`: DataFrame library to use

**Returns:**
- DataFrame with lap timing data

**Columns created:**
- `LapNumber`: Lap number (1-indexed)
- `LapTime`: Lap time in seconds
- `Sector1Time`, `Sector2Time`, `Sector3Time`: Sector times
- `Compound`: Tire compound
- `TyreLife`: Tire age in laps
- `Stint`: Stint number
- `IsPersonalBest`: Boolean flag
- `SpeedI1`, `SpeedI2`, `SpeedFL`, `SpeedST`: Speed trap values
- `TrackStatus`: Track status code
- `Position`: Position at lap end
- `Deleted`: Lap deleted flag
- `DeletedReason`: Reason for deletion
- `FastF1Generated`: Data source flag

### `_process_lap_df`

```python
def _process_lap_df(
    df: DataFrame,
    driver_info: dict[str, Any],
    lib: Literal["pandas", "polars"]
) -> DataFrame
```python Post-process lap DataFrame by adding driver metadata and reordering columns.

**Parameters:**
- `df`: Raw lap DataFrame
- `driver_info`: Driver metadata dictionary
- `lib`: DataFrame library

**Returns:**
- Processed DataFrame with additional columns:
  - `Driver`: 3-letter driver code
  - `DriverNumber`: Car number
  - `Team`: Team name
  - `Time`: Lap timestamp

**Column order:**
The function ensures a consistent column order for compatibility with fastf1:
1. Core identifiers: `Time`, `Driver`, `DriverNumber`, `LapNumber`
2. Timing: `LapTime`, `Sector1Time`, `Sector2Time`, `Sector3Time`
3. Tire info: `Compound`, `TyreLife`, `Stint`
4. Speed traps: `SpeedI1`, `SpeedI2`, `SpeedFL`, `SpeedST`
5. Metadata: `Team`, `Position`, `TrackStatus`, etc.

## Lib Conversion

The pipeline supports both pandas and polars backends:

```python
# Create DataFrame with pandas lib
df_pandas = _create_lap_df(lap_data, lib="pandas")

# Create DataFrame with polars lib
df_polars = _create_lap_df(lap_data, lib="polars")
```yaml

Lib-specific optimizations:
- **pandas**: Uses `pd.DataFrame.from_records()` for efficient construction
- **polars**: Uses `pl.DataFrame()` with schema inference and lazy evaluation

## Data Validation

When `validate_data` is enabled in config, the pipeline validates:

1. **Required fields**: Ensures all required columns are present
2. **Type checking**: Validates data types match schema
3. **Value ranges**: Checks values are within expected ranges
4. **Referential integrity**: Validates driver codes, lap numbers, etc.

**Example validation error:**
```python
InvalidDataError: Invalid lap data at laps/VER/19
  - Missing required field: lap_time
  - Invalid type for sector_1_time: expected float, got str
  - Lap number out of range: 0 (must be >= 1)
```yaml

---

## Performance Considerations

The I/O pipeline is optimized for:

- **Batch processing**: Processes all laps at once rather than one-by-one
- **Lazy evaluation**: Defers expensive operations until data is accessed
- **Memory efficiency**: Uses views and references where possible
- **Type coercion**: Minimizes type conversions

**Typical performance:**
- Process 50 laps: ~5-10ms
- Process 1000 laps: ~50-100ms
- Full session (20 drivers × 50 laps): ~200-300ms

## Internal Implementation

<Accordion title="Column Renaming Strategy">
  The pipeline maintains two sets of column names:
  - **Internal names**: snake_case for JSON keys
  - **Public names**: PascalCase for DataFrame columns

  Renaming happens in `_process_lap_df()` using a predefined mapping dictionary.
</Accordion>

<Accordion title="Type Coercion">
  The pipeline coerces types to ensure consistency:
  - Timestamps → datetime64[ns]
  - Lap times → float64 (seconds)
  - Lap numbers → int64
  - Boolean flags → bool
  - Categorical data → category (pandas) or Enum (polars)
</Accordion>

<Accordion title="Missing Data Handling">
  Missing values are handled gracefully:
  - Numeric fields: NaN (pandas) or null (polars)
  - String fields: empty string or null
  - Boolean fields: False or null
  - Required fields: Raises `InvalidDataError`
</Accordion>
