The `tif1.plotting` module mirrors the core FastF1 plotting data helpers using fastf1 season-aware constants for team and compound colors from 2018 through now. There are few color improvements and extra utilities

Team and driver metadata lookups are session-backed. Exact-match failures raise `KeyError`, invalid colormap values raise `ValueError`, and fuzzy lookup corrections emit warnings instead of silently degrading to legacy fallback tables.

## compatibility

`tif1.plotting` exposes the same plotting-data helper surface documented by FastF1:

- `setup_mpl`
- `get_compound_color`
- `get_driver_abbreviation`
- `get_driver_abbreviations_by_team`
- `get_driver_color`
- `get_driver_name`
- `get_driver_names_by_team`
- `get_driver_style`
- `get_team_color`
- `get_team_name`
- `get_team_name_by_driver`
- `get_compound_mapping`
- `get_driver_color_mapping`
- `list_compounds`
- `list_driver_abbreviations`
- `list_driver_names`
- `list_team_names`
- `add_sorted_driver_legend`
- `set_default_colormap`
- `override_team_constants`

## setup

### `setup_mpl`

```python
def setup_mpl(
    mpl_timedelta_support: bool = True,
    color_scheme: str | None = None,
    *,
    misc_mpl_mods: bool = True,
    **kwargs
) -> None
```

This accepts the FastF1-style call pattern `setup_mpl(False, "fastf1")` and the existing tif1 convenience call `setup_mpl(color_scheme="fastf1")`.

For full timedelta plotting support, install the optional plotting extra:

```bash
uv sync --extra plotting
```

If `timple` is unavailable, tif1 will emit a runtime warning and continue with style-only setup.

```python
import tif1

tif1.plotting.setup_mpl(False, "fastf1")
```

## season-aware colors

Color lookups use the session year when one is available. That means `2018` returns the full hypersoft-to-superhard palette, while `2019+` returns the modern `SOFT`/`MEDIUM`/`HARD` palette.

Team and driver color helpers require `session`. Compound helpers can still be used without a session, but you only get the default modern palette in that mode.

### `get_team_color`

```python
def get_team_color(
    identifier: str,
    session=None,
    *,
    colormap: str = "default",
    exact_match: bool = False
) -> str
```

`colormap` supports `"default"`, `"fastf1"`, and `"official"`. Invalid values raise `ValueError`.

Calling `get_team_color(..., session=None)` raises `ValueError`.

```python
from tif1.plotting import get_team_color

session = tif1.get_session(2024, "Belgian Grand Prix", "Race")
session.load()

ferrari_fastf1 = get_team_color("Ferrari", session, colormap="fastf1")
ferrari_official = get_team_color("Ferrari", session, colormap="official")
```

### `get_compound_color`

```python
def get_compound_color(compound: str, session=None) -> str
```

```python
from tif1.plotting import get_compound_color

session_2018 = tif1.get_session(2018, "Monaco Grand Prix", "Race")


hypersoft = get_compound_color("HYPERSOFT", session_2018)
```

### `get_compound_mapping`

```python
def get_compound_mapping(session=None) -> dict[str, str]
```

```python
compound_colors = tif1.plotting.get_compound_mapping(session)
```

## drivers and teams

### `get_driver_color`

```python
def get_driver_color(
    identifier: str,
    session=None,
    *,
    colormap: str = "default",
    exact_match: bool = False
) -> str
```

Driver colors follow FastF1's current model: both drivers from the same team share the same team color.
Calling `get_driver_color(..., session=None)` raises `ValueError`.

### `get_driver_style`

```python
def get_driver_style(
    identifier: str,
    style,
    session,
    *,
    colormap: str = "default",
    additional_color_kws=(),
    exact_match: bool = False
) -> dict[str, Any]
```

Built-in style generation follows FastF1's team-order rotation for markers and linestyles.

```python
style = tif1.plotting.get_driver_style("VER", ["color", "linestyle", "marker"], session)
```

## legend and overrides

### `add_sorted_driver_legend`

```python
def add_sorted_driver_legend(ax, session, *args, **kwargs)
```

This groups drivers by team and keeps teammates in the same order used for style selection.

Unresolved legend labels are preserved and appended after all resolved driver labels.

### `set_default_colormap`

```python
def set_default_colormap(colormap: Literal["fastf1", "official"]) -> None
```

### `override_team_constants`

```python
def override_team_constants(
    identifier: str,
    session,
    *,
    short_name: str | None = None,
    official_color: str | None = None,
    fastf1_color: str | None = None
) -> None
```

Overrides are session-scoped and use exact-match semantics.

```python
tif1.plotting.override_team_constants(
    "Ferrari",
    session,
    short_name="Scuderia",
    fastf1_color="#ff0000",
)
```

## examples

```python
import matplotlib.pyplot as plt
import tif1

tif1.plotting.setup_mpl(False, "fastf1")

session = tif1.get_session(2024, "Monza", "R")
session.load()
laps = session.laps.pick_drivers(["VER", "LEC"])
colors = tif1.plotting.get_driver_color_mapping(session)

fig, ax = plt.subplots(figsize=(12, 6))
for driver in ["VER", "LEC"]:
    driver_laps = laps[laps["Driver"] == driver]
    ax.plot(
        driver_laps["LapNumber"],
        driver_laps["LapTime"],
        color=colors[driver],
        label=driver,
    )

tif1.plotting.add_sorted_driver_legend(ax, session)
ax.set_xlabel("Lap number")
ax.set_ylabel("Lap time")
plt.show()
```
